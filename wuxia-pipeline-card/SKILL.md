# Wuxia Pipeline Card — 武侠流程卡·从0到1

> **定位**：用武侠风连环小图，把一件事情从0到1的完整流程讲得清清楚楚。
> - 不是概述、不是原理，而是**可执行的步骤过程**
> - 每张卡片以「连环画」形式展开：上一张图引出下一张图，串联成完整武侠故事线
> - 输出：本技能负责内容策划+配图生成 → HTML制卡委托 `@skill://顶尖的前端产品` → `@skill://前端专家级工程师` 完成 →（可选）PNG 长图 + 切割 + 推公众号

## 内容结构（五章流程）

### 壹回 · 起手式 — 你要做什么？

- **目标导向**：一句话说清要完成什么
- **前置准备**：需要什么工具/环境/材料（明确清单）
- **预期耗时**：给读者心理预期
- **成果预览**：做出来是什么样

### 贰回 · 招式图谱 — 全流程纵览

- **流程总览图**：全步骤一览（核心组件 `.pipeline-overview`）
- 每一步一个小图 + 步骤名，读者扫一眼就知道总共多少步、每步做什么
- 像秘籍目录一样清晰

### 叁回 · 逐招拆解 — 一步一图一步法

- **对每一步**展开详解（核心组件 `.step-detail`）：
  - 步骤编号 + 名称
  - 具体操作（命令/动作/操作）
  - 为什么要这样做（原理）
  - 墨小颠配图（正在执行此步骤的武侠场景）
  - 常见陷阱警告（`.trap-box`）

### 肆回 · 运功心法 — 关键诀窍

- 汇总全流程**最容易卡住的3-5个点**，给口诀级解法
- 加速技巧：省时间的骚操作
- 调试心法：出问题了怎么排查

### 伍回 · 功成圆满 — 检查清单 + Next Step

- **完成确认**：怎么判断做对了
- **下一步**：做完了这个，接下来可以做什么（引导留存）
- **互动引导**：留言交流、分享成果

## 视觉风格

延续武侠卷轴/宣纸基底，流程卡专属组件：

| 组件 | 作用 | 视觉效果 |
|---|---|---|
| `.pipeline-overview` | 流程全貌一览图 | 竖向步骤面板，每步一个武侠小图 + 标题 |
| `.step-detail` | 单步拆解 | 详细说明、操作要点、配图 |
| `.pipe-arrow` | 步骤间连接 | 朱砂红箭头/墨线连接，体现武侠"一招接一招" |
| `.step-num` | 步骤编号 | 书法大字编号，红底金边 |
| `.tool-tag` | 工具/命令标签 | 墨底金字的操作标签，像武林秘籍口诀 |
| `.trap-box` | 陷阱警告 | 每一步可能踩的坑，提前警示 |

### 连环画配图策略

每一张配图要让墨小颠**不是站桩看客，而是真的在干这件事**：

| 步骤阶段 | 墨小颠在干什么 | 画风 |
|---|---|---|
| 起手式（准备） | 翻阅秘籍、磨墨、布阵 | 静谧、蓄势待发 |
| 第一招（入门） | 挥剑出鞘、运功、起手 | 动感、速度线 |
| 中间招（进阶） | 实战对练、拆招、闪避 | 激烈、火花四溅 |
| 杀招（关键） | 一剑封喉、冲破关隘 | 爆发、金光炸裂 |
| 收功（完成） | 收剑入鞘、打坐、满意微笑 | 宁静、功成圆满 |

## 生成流程

### 1. 选择主题

如果用户没指定，主动询问或挑选一个常见的流程类主题（如 CI/CD 搭建、项目部署、环境配置等）。

### 2. 资料校验

通过 WebSearch / WebFetch 至少 2 个权威来源，确认流程步骤、命令、工具版本准确无误。

### 3. 内容构建

按五章结构撰写内容（非 `是什么/为什么/怎么用/有什么问题` 四维度），每章填入模板占位符。

### 4. 生成墨小颠配图

调用 `@skill://wuxia-avatar` 生成手绘线稿风格配图。配图数量：`2（起手+收功）+ 1（总览连环画）+ N（步骤数）`。

配图生成流程（5a→5c）：

**4a. 观文 → 配图策略**：读完内容后，提炼每个步骤的核心场景：

- **起手图**：墨小颠在翻阅秘籍、磨墨准备 → "这仗要怎么打"
- **流程连环画**：全家福/全流程俯瞰 → 多格拼接或全景俯瞰图
- **步骤图 × N**：每步墨小颠在执行操作 → 挥剑出鞘/实战对练/冲关
- **收功图**：墨小颠满意收剑 → 功成圆满的宁静画面

**4b. 落笔 → 生图**：用 `image_gen` 逐张生成，参数：`size: "1024x1024"`，`style: "vivid"`，`output_dir: "generated-images/"`，文件名 `xiaodian_pipeline_{slug}_{序号}.png`。

必检要素（每张生成前确认）：
- 乱毛 bun + 朱砂红葫芦（角色锚点到位）
- 墨小颠在干技术相关的活（不是站桩看客）
- prompt 首行写 `Chinese text ONLY. All text characters must be Chinese.`
- 批注 2-5 个词，汉字清晰可读、笔画完整

**4c. 验稿**：生成后逐字默念批注汉字，确认无缺笔画/假名/英文/粘连。不合格 → 重生成。

## 运营策略（涨粉导向）

### 标题情绪设计（流程卡专属）

流程卡天然适合**获得感 + 紧迫感**组合：

| 情绪类型 | 示例标题 |
|---|---|
| 获得感 | "从零到一，用Docker部署全栈项目（全流程图解）" |
| 紧迫感 | "2026年了，还不会这个流程？一图搞定" |
| 好奇心 | "高手是怎么配置开发环境的？12步全流程解密" |
| 社交货币 | "面试官让我现场搭一套CI/CD，我掏出了这张图" |

### 标题公式速查（实战胜率高的模板）

| 编号 | 公式模板 | 结构拆解 |
|---|---|---|
| **F1** | 搞懂`{X}`，`{领域}`你就横着走 | 获得感 → 狂妄结果 |
| **F2** | 被`{X}`虐哭后，我悟了这套`{比喻}` | 焦虑共鸣 → 解法承诺 |
| **F3** | 面试官问`{X}`，我掏出`{独家武器}` | 社交货币 → 反杀场景 |
| **F4** | 没用过`{X}`的`{岗位}`，`{严重后果}` | 紧迫感 → 身份威胁 |
| **F5** | 把`{X}`画成`{画面}`，`{对象}`一看就懂 | 好奇心 → 认知降维 |
| **F6** | 花了`{N}`小时搞懂`{X}`，现在`{N}分钟`讲给你听 | 时间投入 → 读者占便宜 |
| **F7** | `{N}`个问题，带你彻底搞懂`{X}` | 结构清晰 → 获得感可量化 |

**标题铁律**：
1. 禁止说明书式标题（"Docker 部署流程图"）
2. 禁止空洞情绪词（"惊天揭秘""99%的人不知道"）
3. 必须对位读者身份，≤ 32 字
4. 每个标题提供 3 个候选，默认取第1个

## 内容风格指南

- 用第一人称「我」口吻，像脱口秀演员在讲段子
- 每个类比要具体、可触摸，避免抽象术语堆叠
- 技术术语必须准确，错误处应及时纠正
- 代码片段必须可执行真实命令，不要写伪代码
- 警告框必须标注具体风险，不能泛泛而谈

---

## 常见错误避免

- **概念别混淆**：相近技术名词必须厘清边界，不要张冠李戴
- **别绝对化安全/性能**：避免"比 XXX 快 10 倍"等无数据支撑的断言
- **别忽略反作用条件**：技术优势往往带有代价，必须点出适用边界
- **类比要贴切**：一个技术只找一个最贴切的类比，不要生搬硬套
- **写代码遵循阿里代码规范，禁止代码使用表情**
- **运营禁忌**：标题不要过度标题党，情绪调动要真实

本技能是「武侠流程卡」的**内容策源地**，HTML 编码环节按以下管线流转：

```
wuxia-pipeline-card（本技能：内容策划 + 配图生成）
        │
        │  交付：五章内容 + 武侠配色表 + 模板路径 + 配图清单
        ▼
顶尖的前端产品（产品经理：需求分析 → PRD + config.json）
        │
        │  交付：PRD.md + config.json（含武侠风色彩/字体/组件约束）
        ▼
前端专家级工程师（编码实现：读取 config.json + 武侠模板 → 生成 HTML）
        │
        │  交付：移动端 HTML + 网页版 HTML → output/{名词}/
        ▼
wuxia-pipeline-card（接收验证 + 最终交付）
        │
        │  （可选）
        ▼
wechat-publisher（截图 + 切割 + 推送公众号）
```

### 各角色职责边界

| 角色 | 负责 | 不负责 |
|------|------|--------|
| **本技能** | 内容策划、步骤拆解、武侠类比、墨小颠配图生成、资料校验 | HTML 编码、CSS 编写 |
| **顶尖的前端产品** | 需求分析、产品架构、PRD 输出、config.json 生成、派发指令 | 具体编码实现 |
| **前端专家级工程师** | 读取模板+配置、填充内容、生成 HTML/CSS/JS、写入文件 | 内容文案创作、配图生成 |

### 交接标准

本技能向 `@skill://顶尖的前端产品` 交付时必须包含：

1. **五章完整内容**（已校验、已润色）
2. **武侠配色表**（宣纸底/墨色三层/朱砂/金/玉/墨线 → 完整 HEX）
3. **模板文件绝对路径**（card_template.html + card_template_web.html）
4. **样式文件绝对路径**（style.css + web.css）
5. **配图文件路径清单**（按步骤排列）
6. **输出目录**（output/{技术名词}/）
7. **文件命名规则**（`{名词}_{时间戳}_pipeline.html` / `_pipeline_web.html`）
