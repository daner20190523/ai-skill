# Xiaohongshu AI Art Cards - 小红书AI绘画同色系连环画卡片

## 核心理念

将 AI 生成的绘画作品转化为小红书**多图轮播帖**——**不是单幅晒图，而是用 5 幅同色系、有故事线的画作构成一个微型视觉展览**。每张卡本身是一幅独立的 AI 画作，5 张连起来是一部可以"翻"的视觉短篇。

**涨粉的底层逻辑不再是"利他提示词"——漫画赛道已经卷成红海。真正的增长点在"视觉辨识度×叙事密度"**：

```
主题概念 -> 拆解5个叙事节点 -> 生成5幅同色系AI画作 -> 1个HTML -> 切图 -> 5张PNG
                  |                           |                      |
                  v                           v                      v
            故事线设计                   一组prompts.md           每张都是画作
            (起·承·转·合·余)          (单文件存档，可复用)        (不做纯色底色)
```

> **新红线：每张卡片都必须是 AI 生成的画作。** 以前的"Card 1 画作 + Cards 2-5 纯色底色"已经过时——用户翻到第二张看到纯色文字卡就划走了。现在小红书用户的「视觉预期」是：翻一张是一幅新画。5 张画作必须满足四个条件：(1)**同色系**——5 幅画共享一个色调系统，像同一个调色盘画出来的；(2)**有故事线**——5 幅画之间是起承转合的关系，不是 5 幅无关的画拼在一起；(3)**每幅画可独立欣赏**——单张拿出来也是一幅完整的作品；(4)**人物一致**——如果画作中涉及人物，5 幅画中的角色必须是同一个人：相同的五官特征、相同的发型发色、相同的妆容风格、相同的服饰（或同一身衣着的合理变化）、相同的体型身段，绝不能让读者觉得"换了个人"。

## 创作灵魂：连环画叙事哲学

本技能的核心竞争力不是"生成一张好看的画"，而是**用 5 幅同色系的 AI 画作讲一个视觉短篇**。每幅画是故事的一个章节，5 幅连起来是一部静默的电影。

### 五幕故事线框架

每期必须设计一条 5 幕故事线，5 幅画分别对应一个叙事节点：

```
第一幕 [起]  →  第二幕 [承]  →  第三幕 [转]  →  第四幕 [合]  →  第五幕 [余]
    ↓               ↓               ↓               ↓               ↓
 确立主题        深化/展开       意外/转折       碰撞/高潮       落幕/回味
 第一眼被抓住    情绪堆积        出乎意料        矛盾顶点释放   什么都没发生的诗意
```

**故事线设计原则**：

| 原则 | 说明 | 示例（主题：海底遗书） |
|------|------|---------------------|
| **同一色系** | 5 幅画必须使用同一套色调系统——像同一个画家的同一个系列 | 深蓝底+磷光绿+旧纸白贯穿全 5 幅 |
| **主体延续** | 核心意象在 5 幅画中都出现，但姿态/状态递进 | 第1幅：一封信沉入水中 → 第3幅：信纸散开，文字变成鱼群 → 第5幅：空信封浮上水面 |
| **情绪爬坡** | 情绪不能平——必须从第一幅到第四幅逐步爬升，第五幅释放 | 静谧→好奇→骚动→狂喜/崩塌→释然 |
| **单幅完整性** | 每一幅拿出来都是成立的独立画作，不是"前一幅的补充说明" | 第3幅即使不看前后，也是一幅完整的"文字变成鱼群"超现实画作 |
| **人物一致性**（涉人物时） | 5 幅画中出现的角色是同一个人——五官、发型、妆容、服饰、体型、肤色六个维度保持一致 | 同一个人在第1幅穿白衬衫坐沙发，第3幅仍然穿同一件白衬衫在桌前画画——读者一眼认出"还是他" |

### 同色系视觉统一法则

5 幅画的提示词中必须共享以下参数：

```
统一色系:   dominant colors = [同一组 2-3 个主色]
统一光源:   lighting direction / time of day = [同一个光源条件]
统一风格:   art style = [同一个艺术流派/技法]
统一质感:   texture / medium = [同一种材质感：油画/水墨/数字厚涂/浮世绘]
统一角色:   character description = [同一人物的五官/发型/妆容/服饰/体型固定描述]（涉人物时必须）
可变部分:   composition / subject pose / focal point（构图和主体姿态在变，推动叙事）
```

> **人物一致性铁律（涉人物时必须遵守）**：5 个 prompt 中必须在最前面插入一段**固定的人物描述块**——完全相同的字符，逐字照搬，不增不减。这段描述覆盖六个维度：
> - **五官**：脸型、眼型、鼻型、嘴型、肤色（如 "oval face, almond eyes, straight nose, light skin tone"）
> - **发型**：发长、发色、发型（如 "short black hair, side part"）
> - **妆容**：眉形、唇色、有无眼镜等（如 "thin eyebrows, natural lip color, no glasses"）
> - **服饰**：上衣、下装、鞋子的颜色和款式（如 "white button-up shirt, black trousers, brown leather shoes"）——5 幅画中可以合理换装（如脱了外套），但不能换风格
> - **体型**：身高、胖瘦、肩宽（如 "slim build, average height, narrow shoulders"）
> - **场景延续**：如果 5 幅画都在同一场景（如同一间房），场景描述也必须一致（同一扇窗、同一张沙发、同一面墙）

### 提示词美学原则（单幅）

| 原则 | 说明 | 示例 |
|------|------|------|
| **概念先行** | 每幅画先有一个哲学内核，再找意象表达 | "遗忘不是消失，是沉到看不见的地方" → 一封写满字的信缓缓沉入深海 |
| **线条清晰** | 轮廓分明、结构利落——不要模糊柔焦，要有"一笔就是一笔"的果断 | 清晰的人物轮廓 / 利落的建筑边缘 / 植物的叶子片片分明 |
| **色彩鲜明** | 色块明确、对比干净——不灰不脏不暧昧 | 主色+辅助色+点缀色，三色体系干净利落 |
| **矛盾共生** | 画面中植入对立元素，制造张力 | 深海中的一束光 / 废墟正中一朵盛放的红色花 |
| **留白是节奏** | 5 幅画总体构图偏满，但第五幅必须留白收尾 | 前面4幅信息密度高，第5幅大面积天空/水面/空旷 |

### 禁止的提示词套路

- ❌ "4K, 8K, ultra detailed, masterpiece"——空洞的技术标签堆砌
- ❌ "beautiful, stunning, gorgeous"——没有信息量的形容词
- ❌ "trending on ArtStation"——没有灵魂的流行词
- ❌ 纯写实风景而无叙事张力——没有冲突就没有故事
- ❌ 5 幅画色系不统一——杜绝"一张金色一张蓝色一张黑白"的拼凑感
- ❌ 涉人物时 5 幅画角色不一致——杜绝"第一张是短发男、第三张变成长发女"的崩坏感

## 四大涨粉引擎（策展人式视觉叙事）

### 引擎 1：视觉辨识度（权重最高——新赛道核心）

AI 绘画赛道已经从"提示词利他"卷成了红海。现在真正稀缺的是**一眼能认出是谁的画**——视觉辨识度比提示词分享更值钱。

| 辨识度维度 | 落地方式 | 效果 |
|-----------|---------|------|
| **固定色系签名** | 每期 5 幅画共享同一套色调——用户看到这个配色就认出是你 | "这个色调一看就是 XXX 的画" |
| **固定叙事节奏** | 每期都是 5 幕结构：起承转合余——用户每次都期待"下一格会发生什么" | 追漫式的持续性关注 |
| **固定画风标签** | 明线清晰 + 色块果断 + 构图干净——三要素贯穿所有期 | 建立"XX风AI绘画"品类认知 |
| **固定 Card 1 右下署名** | 每期 Card 1 固定的署名排版 | 滚动时一眼认出的品牌锚点 |

> **新原则：提示词可以学走，视觉辨识度学不走。** 搭建属于自己的"视觉语法"才是壁垒。

### 引擎 2：叙事钩子（驱动翻页——替代争议评论）

多图轮播帖的核心行为是**翻页**。如果用户只翻 1-2 张就划走，系统判定为"吸引力不够"。替换策略：

```
旧策略（文字争议钩子 Card 4）："你觉得AI画的梵高还是梵高吗？"
新策略（视觉翻页钩子，贯穿全部 5 张）：

  Card 1 → Card 2：第一幅给一个"然后呢？"的悬念
    示例：一封沉入水中的信 → 第二幅：信纸散开了，文字在发光
  
  Card 2 → Card 3：第二幅积累情绪 → 第三幅来一个视觉反转
    示例：文字在海底发光 → 第三幅：文字突然变成鱼群，向四面八方游去
  
  Card 3 → Card 4：第三幅的"意外" → 第四幅推到极致
    示例：鱼群四散 → 第四幅：整个海底被金色文字照亮，像一场无声的爆炸
  
  Card 4 → Card 5：第四幅的"高潮" → 第五幅退潮、留白、回味
    示例：金色文字爆炸 → 第五幅：水面恢复平静，只剩一个空信封浮在月光下
```

**视觉翻页钩子公式**：`前一幅画中埋下一个"变化的种子" → 下一幅画中这个种子"开了"`

### 引擎 3：人设锚点（存图→存画风）

用户的关注动因是"这个人的画风我喜欢，下次还想看"。在卡片和正文中强制植入人设信号：

| 人设信号 | 落地方式 | 效果 |
|---------|---------|------|
| 创作理念标签 | 正文第一句 | "这是同色系连环画的第X期——5幅画讲一个视觉短篇" |
| 固定署名排版 | Card 1 右下角 + Card 5 右下角统一署名样式 | 翻到最后一页知道的"署名"，翻回第一页认出的"签名" |
| 视觉签名 | 所有卡片同色系——用户不用看文字就知道是谁 | 刷feed时一眼认出的"色感" |
| 系列归属 | 正文末尾 | "『同色系视觉短篇』第X期" |
| 画风标签 | 正文底部固定画风标签 | "#线条清晰 #同色系连环画 #AI叙事绘画" |

### 引擎 4：搜索截流（长尾视觉流量）

小红书 **68% 流量来自搜索**。标题和正文嵌入用户会搜的关键词：

**标题关键词公式**：`[主题概念] + [视觉风格] + [故事钩子] + [5幅]`

```
实例：
❌ "今天用AI画了一组好看的画"
✅ "一封沉入深海的遗书|5幅AI画作讲一个故事|同色系视觉短篇"
```

**正文底部嵌入长尾搜索词**（自然行文，不堆砌）：
```
"如果你也喜欢有故事感的AI画作" / "5幅画看完一个故事" / "同色系AI绘画系列"
```

## 卡片结构（5张同色系连环画）

**核心变化**：每张卡都是一幅独立的 AI 画作（`bg_01.png` ~ `bg_05.png`），不再是"Card 1 画作 + 其余纯色"。5 张画构成「起承转合余」视觉短篇。

### Card 1 — 起：第一眼（确立世界）

```
+--------------------+
|                    |
|   [AI画作 全幅]    |
|   [画面：故事的开端]|
|   [画面占95%面积]  |
|                    |
|   [底部半透明遮罩] |
|   [标题/旁白 以讲清楚为准] |
|   [-- 智韵阁]      |
|                    |
+--------------------+
```

**设计要求**：
- AI 画作 `bg_01.png` 全幅铺底，画面占 95% 面积
- 底部保留 120px 半透明渐变遮罩，内放白色文字
- 一行标题：像画展标签，不解释画面，只是一个名字
- 右下角署名 `-- {{AUTHOR_NAME}}`，字号 22px 白色半透明
- **画面要求**：确立这个系列的世界观——光线、色彩、核心意象第一次登场。像电影开场的第一个全景镜头

### Card 2 — 承：靠近（情绪堆积）

```
+--------------------+
|                    |
|   [AI画作 全幅]    |
|   [画面：更近一步] |
|   [同一意象，拉近] |
|                    |
|   [底部半透明遮罩] |
|   [一行情境文字]   |
|                    |
+--------------------+
```

**设计要求**：
- AI 画作 `bg_02.png` 全幅铺底
- 与 Card 1 同色系，但在构图上"更近一步"——视距拉近、细节放大
- 底部文字描述一个正在发生的状态（像电影的"第二镜头"）
- 不做解释，只积累情绪
- **画面要求**：让观众"走进去"——从 Card 1 的远景或全景，推进到中景。同一意象但给更多细节

### Card 3 — 转：意外（视觉翻转）

```
+--------------------+
|                    |
|   [AI画作 全幅]    |
|   [画面：出人意料] |
|   [意象开始变化]   |
|                    |
|   [底部半透明遮罩] |
|   [一行反转文字]   |
|                    |
+--------------------+
```

**设计要求**：
- AI 画作 `bg_03.png` 全幅铺底
- **这一幅是翻页率的关键**——画面必须有一个视觉意外：核心意象开始发生超现实的变化
- 底部文字点破这个意外，以讲清楚为准
- **画面要求**：前两幅建立的世界观在这一幅发生偏转。什么东西裂开了、散开了、发光了、开始飘动了——视觉上的"变化瞬间"

### Card 4 — 合：绽放（能量顶点）

```
+--------------------+
|                    |
|   [AI画作 全幅]    |
|   [画面：推向极致] |
|   [变化到达顶点]   |
|                    |
|   [底部半透明遮罩] |
|   [一行态度文字]   |
|   [-- 智韵阁]      |
|                    |
+--------------------+
```

**设计要求**：
- AI 画作 `bg_04.png` 全幅铺底
- **这一幅是视觉密度的顶点**——信息量最大、色彩最饱和、动静对比最强烈
- 底部文字一句话态度——可以有锋芒，是 5 幅中最"敢说"的一句
- 右下角署名 `-- {{AUTHOR_NAME}}`
- **画面要求**：Card 3 的变化在 Card 4 推向极致——像交响乐的第四乐章。如果 Card 3 是"文字开始发光"，Card 4 就是"整片海底被金色文字照亮"

### Card 5 — 余：退潮（终结日）

```
+--------------------+
|                    |
|   [AI画作 全幅]    |
|   [画面：落幕/留白]|
|   [大面积空镜]     |
|                    |
|   [底部半透明遮罩] |
|   [底部 3个变体方向]|
|   [一行收束文字]   |
+--------------------+
```

**设计要求**：
- AI 画作 `bg_05.png` 全幅铺底
- **这一幅必须大面积留白**——画面中至少 40% 是天/水/空/光/雾（留白元素）
- 底部列出 3 个同画风可执行变体方向（延续利他基因）
- 底部收束文字，像合上书的那句
- **画面要求**：第四幅的激烈退潮了。什么都不发生，但什么都发生过了。像电影落幕后的那个空镜

### 5 幅画统一设计规则

| 规则 | 具体要求 |
|------|---------|
| **同色系** | 5 幅画的 prompt 中必须指定同一组颜色（dominant colors），偏差不超过相邻色相 15° |
| **同光源** | 5 幅画的光源方向、光源类型、一天中的时间保持一致 |
| **同风格** | 5 幅画使用同一种艺术风格描述（如 "Art Nouveau illustration, clear line art"） |
| **同角色**（涉人物时） | 5 幅画中的人物必须使用相同的外貌描述块——五官/发型/妆容/服饰/体型/肤色六个维度一字不改、逐字照搬。场景若重复出现（如同一个房间）也必须描述一致 |
| **同签名** | Card 1 和 Card 4 右下角署名 `-- {{AUTHOR_NAME}}`，字号/位置/透明度完全一致 |
| **底部文字** | 5 张卡底部统一使用半透明渐变遮罩（从 80% 高度开始，底部最深）保护白色文字 |
| **文字以保证故事完整性为准** | 每张卡的底部文字以讲清楚故事为第一优先——字数不设硬上限，但保持简洁不冗余。必要时允许两行甚至三行，宁可多几个字把故事讲完整，也不为了省字让读者看不懂 |

## 策展人式语气基调：像带你逛一场画展

AI 绘画赛道已经从"晒图"进化到"晒视觉叙事能力"。文字不再是主角，画面才是——文字的作用是**架一座桥**，让观众从一张画走进下一张画。语气像一个安静的策展人站在你旁边，**偶尔轻声说一句**，然后退后一步让你自己看。

### 三层文字策略

| 层级 | 作用 | 落点 | 写法 |
|------|------|------|------|
| **命名者** | 给画面一个名字，不给解释 | Card 1 标题 + 每卡底部 | 像给画展标牌写标题：精准、不啰嗦、有诗意但不是故弄玄虚 |
| **引路人** | 在最大转折处出现，点一下就走 | Card 3 底部（唯一有"态度"的文字） | "然后发生了一件事——" / "突然之间" / "就在这时" |
| **合书人** | 落幕时的一句话，轻得像叹息 | Card 5 底部 | 像走出美术馆时跟同伴说的那句：自然而然、不带评价 |

### 代码范例

| 旧（季羡林式/解释式） | 新（策展人式/命名式） |
|---------------------|---------------------|
| "这幅画想说的其实很简单：所有的时间，都可以是同一个瞬间。" | `《日月四季交汇》` |
| "画这幅画的时候，我想起一个人坐在黄昏里什么也没想。" | `某个下午` |
| "第一版画出来完全不对，改了三次提示词才摸到门道。" | （直接删除——不说自己的创作过程） |
| "灵魂不在工具里，在看画人的眼睛里。" | `看就好了` |
| "试试看——不一定每次都对，但方向是好的。" | `第3个变体等你来试` |

### 文字纪律（铁律）

- **故事完整性优先**：每张卡的底部文字不设硬字数上限——讲清楚故事是第一原则。正常 1-2 行，必要时允许 3 行。宁可多几个字把故事讲完整，也绝不为了省字让读者"没看懂这一幕在说什么"
- **不做解释**：文字不解释画面内容——画已经说完了。文字是"旁白"，推动叙事，不是"解说词"
- **不给情绪标签**：不写"孤独""美好""惊叹"——让观众自己感受
- **不写创作心得**：不在卡片里出现"我试了三次""第一版不对""我的想法是"
- **标题用书名号**：Card 1 标题用《》，像画展标牌
- **旁白风格**：像电影里的字幕，在关键画面出现一句话，告诉观众"这里发生了什么"——不需要画面
- **保持简洁**：能一行说完不说两行，但需要两行的时候绝对不能为省字砍成一行

### 仍然保留的季羡林感（仅限正文 content.md）

卡片的文字极度精简了，但 `content.md`（小红书正文框）仍然可以有温度——**正文 150-300 字，季羡林式的诚恳和平静**，像一个人走出美术馆后在门口跟你说两句话。这部分的语气不换。

## AI 连环画生成流程（5 幅同色系）

### Step 0：从主题拆解故事线

用户主题 → 设计 5 幕故事线 → 为每幕确定画面内容 → **提取统一色系和统一风格参数** → **（若涉人物）定义固定角色描述块**

```
示例：
主题"海底遗书"
→ 故事线：沉入 → 散开 → 发光 → 照亮 → 浮起
→ 色系：深海蓝 #0a1a2a + 磷光绿 #4a9070 + 旧纸白 #e8e0c8
→ 风格：Art Nouveau illustration, clear line art, flat colors
→ 光源：水下漫射光，从上到下渐暗
```

**5 幕拆解模板**（强制作业——每次生成前填好此表）：

| 幕次 | 名称 | 画面描述 | 构图变化 | 情绪 |
|------|------|---------|---------|------|
| 1 起 | | | 全景/奠基 | 静谧 |
| 2 承 | | | 推近/中景 | 好奇 |
| 3 转 | | | 特写/变化 | 意外 |
| 4 合 | | | 超广/爆发 | 震撼 |
| 5 余 | | | 远景/空镜 | 释然 |

**角色设定模板**（涉人物时必填——生成前写好，逐字照搬进 5 个 prompt）：

```
角色名：[无明确姓名则用"主角"/"他"/"她"]
五官：脸型/眼型/鼻型/嘴型/肤色
发型：发长/发色/发型
妆容：眉形/唇色/面饰
服饰：上衣/下装/鞋子（颜色+款式）
体型：身高/胖瘦/肩宽
场景：若贯穿同一场景，描述此场景的关键特征
```

### Step 1：为每幕构造提示词

基于 5 幕拆解表，为每幅画构造英文提示词。**必须共享以下统一参数**：

```
统一角色:   character = [固定人物描述块 6 维度]  ← 涉人物时排第一位，5 个 prompt 逐字照搬
统一色系:   dominant colors = [同一组 2-3 色]
统一风格:   art style = [同一个艺术流派描述]
统一光源:   lighting = [同一个光源条件]
统一质感:   medium = [同一种绘画材质/技法]
统一画幅:   --ar 3:4
统一后缀:   固定的负面提示词（no ..., no ...）
```

**5 个提示词的核心差异只在主体状态和构图**，其余全部一致。

**提示词风格格要求**：线条清晰（clear line, defined edges, crisp contours）、色块果断（flat color blocks, bold shapes, no blur）、构图利落（clean composition, graphic quality）。

**人物一致性提示词写法**（涉人物时强制执行）：

每个 prompt 最前面加入完全相同的角色描述块（逐字照搬、一字不改），格式如下：

```
[A person with: oval face, almond eyes, straight nose, thin lips, fair skin tone,
short black hair with side part, thin eyebrows, natural lip color, no glasses,
wearing a white button-up shirt and black trousers,
slim build, average height, narrow shoulders]
→ 然后接本幕独有的场景+动作+构图
```

> **铁律**：不要只写 "a person" 或 "a man/woman"——必须把六个维度的细节全部写进固定描述块。5 个 prompt 的这个块 100% 相同，复制粘贴，不允许任何一个词有变化。

### Step 2：生成 5 幅画作

使用 `image_gen` 工具，逐幅生成（5 次调用，串行）。每幅参数：
- `prompt`: 对应幕次的英文提示词（含统一色系/风格参数）
- `size`: `1080x1440`
- `quality`: `high`

生成后保存为 `bg_01.png` ~ `bg_05.png`。

### Step 3：提取统一配色

从 `bg_01.png` 提取 2-3 个主色，作为 HTML 的 `:root` CSS 变量。**5 幅画使用同一套配色变量**。

### Step 4：保存全部提示词到 `prompts.md`

将 5 幅画的完整创作思路 + 英文提示词统一保存到**一个** `prompts.md` 文件：

```markdown
# {{ART_THEME}}

## 故事线
起 → 承 → 转 → 合 → 余

## 统一参数
- 色系：[色系名] — [色值列表]
- 风格：[艺术流派/技法]
- 光源：[光源描述]
- 反转词：[统一的负面提示词]

## 5 幅画逐幅说明

### 1. 起 — [幕名]
**概念**：[哲学内核，20字]
**画面**：[画面描述]
**提示词**：
```
[英文提示词]
```

### 2. 承 — [幕名]
... [同上格式 x5]
```

**强制性要求**：`prompts.md` 包含全部 5 幅画的提示词，一个文件存档，不可遗漏。

## 正文公式（content.md）—— 150-300字，策展人式 + 季羡林温度

卡片画面已经是 5 幅完整的画作，正文不需要再用文字解释画面。正文的作用是**站在美术馆门口，跟看完画的人聊两句**——不评价画的好坏，只说为什么画了这个。

### 正文结构

```markdown
👋 [人设开场，1句话]：用5幅同色系AI画作讲一个短篇故事——这是第X期。

🎨 系列名：《{{ART_THEME}}》——5幅画，一个故事。

[空行]

[第一段 30-50字] 这个主题的灵感来源——一句诗/一个画面/一个想法。用季羡林式自然的语气，不装。
-> 嵌入长尾搜索词：有故事感的AI画作、5幅画看完一个故事
```

## 系列化内容策略

单篇爆款不会涨粉。**系列是涨粉的唯一加速器**。每篇帖子必须归属于一个明确的内容系列。

### 系列命名公式

```
[数量] + [叙事形式] + [品类]
```

示例：
- `『同色系视觉短篇』`（形式+品类）
- `『5幅画讲完一个故事』`（数量+形式+品类）
- `『一页一幅画』`（节奏+品类）
- `『用AI画连环画』`（工具+形式）

### 正文尾部模板

```markdown
🎨 同系列已发布：「主题1」「主题2」「主题3」-> @主页查看『{{SERIES_NAME}}』合集
```

### 系列落地的 3 个条件

1. **有固定长度**：每期固定 5 幅画——让用户知道翻 5 下就看完了
2. **有固定结构**：起承转合余——用户知道"第 3 张肯定有反转"
3. **有进度**：正文标注"第X期"——让用户感知连续感

## 十五不原则（常见错误避免）

1. **不要 Card 2-5 使用纯色底色**：每张卡都必须是 AI 生成的画作全幅背景
2. **不要 5 幅画色系不统一**：5 个 prompt 必须共享同一组 dominant colors，偏差不超过相邻色相 15°
3. **不要没有故事线就生成**：生成前必须先填好 5 幕拆解表（起承转合余）
4. **不要炫技式提示词**：不堆 "4K ultra detailed masterpiece"，强调 clear lines、flat colors、clean composition
5. **不要让提示词风格割裂**：5 个 prompt 的 art style / lighting / medium 参数必须一致
6. **不要人物不一致**：涉人物时，5 幅画必须先写好 6 维度角色描述块，逐字照搬进每个 prompt 开头——绝不能出现"第一张短发男、第三张变长发女"的崩坏感
7. **不要场景不一致**：贯穿同一场景时（如同一间房），场景描述在 5 个 prompt 中必须相同——同样的窗户、同样的沙发、同样的墙面
8. **不要为了省字砍断故事**：每卡底部旁白以保证故事完整性为第一优先，不计字数——但简洁不冗余，能一行说完不写三行
9. **不要解释画面**：文字是"旁白"不是"解释"——像电影字幕推动叙事，不像导游词
10. **不要缺署名**：Card 1 和 Card 4 右下角必须有 `-- {{AUTHOR_NAME}}`
11. **不要缺变体方向**：Card 5 底部必须列出 3 个可执行变体
12. **不要正文太短**：最低 120 字，150-300 字是黄金区间
13. **不要违规词**：宁可用"挺""比较"也不用"最""第一"
14. **不要文艺腔过重**：正文保留季羡林温度，诚恳自然，不堆大词拉长句
15. **不要把提示词拆成多个文件**：全部 5 幅画的提示词统一保存到 **一个** `prompts.md`，Storyline + 统一参数 + 逐幅说明
