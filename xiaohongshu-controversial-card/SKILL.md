# Xiaohongshu Controversial Cards — 观点分裂卡片生成（桥接层）

## 技能定位

本技能是 **「观点分裂文案」与「前端顶尖程序员」之间的桥接层**。它不做最终渲染、不写生产 CSS、不调用切图——它把一篇争议文案**结构化拆解为卡片内容块 + 变量映射 + 可交付模板**，让前端工程师能直接接手。

```
观点分裂文案 (controversial-opinion-writer)
   ↓ 产出：争议性爆款文案 Markdown
   ↓         （双方观点 + 金句 + 作者孔庄洞见 + 互动）
本技能 (xiaohongshu-controversial)   ← 你在这里
   ↓ 拆解：5 张卡片内容块
   ↓ 映射：{{VARIABLE}} 变量表
   ↓ 产出：cards.html 模板 + config.json/yaml + DELIVERABLE.md
前端顶尖程序员 (前端顶尖程序员)
   ↓ 实现：CSS/JS/IMG 分层 + 真实切图
小红书 5 图轮播帖
```

> **红线：本技能只产出「内容 + 结构 + 占位模板」，不写最终样式。** 所有视觉实现、响应式、切图脚本调用都交给「前端顶尖程序员」——它的输出目录、CSS 分层规范、config.js 加载器才是真相来源。

## 输入格式

「观点分裂文案」技能产出的 Markdown，标准结构：

```
# [吸睛标题]
> 💡 Trending: #[话题]# ● [平台] ● [热度数据]

[观点A：强情绪，含 1 句金句]
[自然转折：But wait...]
[观点B：对立视角，含 1 句金句]

## 我的看法
[孔子/庄子融合的洞见]

## 你怎么看？
[站队互动]
```

若上游输出不完全符合此结构，按下面的「卡片拆解映射」强行归位——缺失的块留空字符串 `""`，不杜撰。

## 卡片拆解映射（5 张 · 1080x1440）

| 卡片 | 来源区块 | 卡片职责 | 必含元素 |
|------|---------|---------|---------|
| **Card 1 封面钩子** | 标题 + 💡热点 + 开场钩子 | 第一眼抓住，亮明争议话题 | 大字标题 · 热点标签 · 一句钩子 |
| **Card 2 观点A** | 观点A 段 | 先立一方，让读者点头 | 立场标签 · 观点A正文 · 金句高亮 |
| **Card 3 转折** | 自然转折句 | 视觉/情绪支点，制造"但是…" | 转折短句 · 左右分屏暗示两方 |
| **Card 4 观点B** | 观点B 段 | 反转，理解对立面 | 立场标签 · 观点B正文 · 金句高亮 |

## 背景图配图（AI 生图步骤）

每张卡都有一个背景图层 `.card-bg` + 压暗遮罩 `.card-mask`，让文字始终可读。本技能负责**生成背景图并填入 `{{CARDn_BG}}` 变量**；若不生图，变量留空 `""`，模板自动降级为纯色/渐变底（不影响交付）。

### 配图原则

- **服务于文字，不抢戏**：背景是氛围，不是主角。选**大面积留白、低对比、氛围感**的意象，避免密集主体、强文字、人脸特写。
- **弱化处理**：所有背景图叠 `.card-mask` 遮罩（顶 35% → 底 80% 渐暗），确保正文/金句白字清晰。
- **同系一致**：5 张背景图共用同一视觉母题 + 同一配色（对应 `{{PALETTE_NAME}}`），保证轮播连贯。
- **叙事呼应**：封面(1)最有张力，观点A(2)/观点B(4)可用**冷暖/明暗对立**暗示两方，转折(3)取"分岔/交叉"意象，收口(5)归于平和留白。
- **尺寸**：竖版 `1024x1536`（贴近 1080×1440 比例，切图时 `cover` 裁切）。

### 生图提示词模板

按议题 + 配色填空，调用 `image_gen` 逐卡生成：

```
{主题氛围意象}, {配色关键词}, 极简构图, 大面积负空间/留白,
柔和光影, 低对比, 氛围感, 无文字, 无人脸特写,
适合作为竖版海报背景, 电影感, 高级质感
```

配色关键词对照（承接配色表）：

| 方案 | 生图配色关键词 |
|------|---------------|
| 墨锋 | deep charcoal black, crimson red accents, ink wash |
| 朱砂 | dark vermilion, champagne gold, oriental mood |
| 暮金 | warm brown black, amber glow, dusk tone |
| 极夜 | near-black, moonlight white highlights, minimal |
| 深海 | deep navy black, cold blue accents, abyss |
| 春醒 | deep green, soft daylight, natural texture |
| 素宣 | warm rice-paper beige, ochre brown, xuan paper |

> 示例（买房·暮金·封面）：`empty modern apartment at dusk, warm brown black and amber glow, minimal composition, large negative space, soft light, low contrast, atmospheric, no text, no face closeup, vertical poster background, cinematic, premium quality`

### 生图流程

1. 逐卡按议题写提示词（母题一致、配色一致）
2. 调 `image_gen`：`size=1024x1536`，`output_dir=output/{主题}_{时间戳}/images`，文件名 `card1_bg.png`…`card5_bg.png`
3. 把路径写进 `config.json` 的 `backgrounds` 段与模板 `{{CARDn_BG}}`
4. 预览确认文字在遮罩下可读；不达标则调提示词重生或改遮罩浓度（前端可调 `.card-mask` 透明度）

> **成本意识**：背景图非必需。若用户只要文字卡片，跳过本章、变量留空即可。生图仅在用户要求"配图/带背景/更有质感"时执行。

生成交付包前逐卡扫描，命中立即替换：

### 绝对禁止词

| 类别 | 禁止词 | 替代 |
|------|--------|------|
| 极限词 | 最/第一/唯一/绝对/顶级/百分百 | 很/挺/比较/相对 |

| 诱导词 | 加微信/私信我/关注我/扫码 | 评论区聊聊/常来看看 |

| 虚假数据 | XX万人/全网爆火/疯了 | 不少人/蛮受欢迎 |

### 灰色地带（慎用）

- "必看""保姆级" 每篇 ≤1 次
- emoji 每卡 ≤2 个
- 争议表达要有态度但不过线（不人身攻击、不煽动对立）

## 执行流程

1. **接收文案**：读取「观点分裂文案」产出的 Markdown（或用户粘贴的文本）
2. **拆解归位**：按「卡片拆解映射」表把文案切成 5 块，缺失块填空字符串
3. **填配置**：把所有内容写入 `config.json` / `config.yaml`，键名严格对应变量表
4. **选配色**：按议题初选 `{{PALETTE_NAME}}`，写入配置
5. **配背景图（可选）**：若用户要"配图/带背景/更有质感"，按「背景图配图」章节逐卡 `image_gen` 生成到 `images/`，路径写入 `backgrounds` 段与 `{{CARDn_BG}}`；否则变量留空降级纯色
6. **出模板**：复制 `templates/cards.html`，确认 `{{VARIABLE}}` 与 `{{CARDn_BG}}` 占位符齐全（内容由前端用 config 注入，或人工替换）
7. **写交付说明**：`DELIVERABLE.md` 列出变量清单、配色变量、背景图提示词、切图命令（`python scripts/html_slice.py output/{主题}_{时间戳}/cards.html --card-height 1440`）
8. **交前端**：提示用户「交给前端顶尖程序员技能，输出目录见 project.yaml」

## HTML 模板规范（给前端工程师的约束）

模板 `templates/cards.html` 硬约束（前端实现时必须遵守）：

- 每张 `.card` 宽高固定 `width:1080px; height:1440px`
- 卡片间**无间距**（margin:0），紧贴堆叠成 `.card-stack`
- `body { margin:0 }`，完全无白边；`box-sizing: border-box` 全局
- 所有文字用 `px` 单位，不随视口缩放
- 每卡含 `.card-bg`（背景图层，`background-image` 由 `{{CARDn_BG}}` 注入）+ `.card-mask`（压暗遮罩）；内容层 z-index 必须盖在其上
- 背景图缺省时 `.card-bg` 无图，卡片降级为 `var(--bg)` 纯色/渐变，不得报错或留白
- `.card-mask` 遮罩浓度前端可调，但须保证白字正文/金句在任意背景图上可读
- 金句用 `<mark>` 或独立 class 高亮，不写死颜色，用 `var(--accent)`
- `-- {{AUTHOR_NAME}}` 出现在 Card 1 左下、Card 5 底部
- 不要阴影/边框/渐变文字/动画

## 十不原则

1. **不写最终样式**：只给 `{{VARIABLE}}` 占位 + CSS 变量名，视觉交给前端
2. **不杜撰内容**：卡片文字全部来自上游文案，缺失块留空 `""`
3. **不超 5 卡**：5 是小红书轮播黄金数，不增不减
4. **不缺人设锚点**：Card 1 左下 + Card 5 底部必须有 `-- {{AUTHOR_NAME}}`
5. **不缺站队互动**：Card 5 必须有二选一站队钩子（驱动评论区）
6. **不缺双引文**：Card 5 必须同时有孔子 + 庄子引文（作者洞见的灵魂）
7. **不写死颜色**：模板只用 `var(--xxx)`，配色由前端按 `{{PALETTE_NAME}}` 落地
8. **不违规词**：生成交付包前逐卡扫描极限/承诺/诱导词
9. **不调切图脚本**：切图是前端职责，本技能只写 `DELIVERABLE.md` 告知命令
10. **不丢溯源**：`source_article.md` 必须随包存档，方便前端回查原文
11. **背景图不抢戏**：背景图必须大留白、低对比、无文字无人脸，且叠遮罩保证文字可读；不生图时变量留空降级纯色，不报错
