# Wuxia Card — 武侠漫画风技术知识卡片

## 触发条件

- 用户提到"武侠""秘籍""漫画""江湖""武功""夸张""刺激""热血"
- 用户说"把 XXX 讲成武侠秘籍""用武侠风解释 XXX"

## 工作模式

本技能是**风格模块**，只负责**武侠风专属配置**（视觉风格 + 组件规范 + 内容要素）：

- **内容生成**委托给 `@skill://tech-knowledge-card` 的总流程（WebSearch 校验→脱口秀专家内容构建→墨小颠配图→Blog 同步）。
- **HTML 制卡**委托给 `@skill://顶尖的前端产品` → `@skill://前端顶尖程序员` **从零实现模板**。本技能**不内置任何 `.html` / `.css` 模板文件**，只交出视觉规范与组件清单，由前端程序员按规范实现移动端（420px）与网页版响应式两套 HTML。

## 变量约定

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `{{AUTHOR_NAME}}` | `daner技术栈` | 作者署名 |
| `{{QR_CODE_IMAGE}}` | `../../assets/qrcode_wechat.jpg` | 二维码路径（相对 output/wuxia/{技术名词}/） |

## 生成流程

1. **转发内容生成**：将技术名词和"wuxia"风格标识传递给 `@skill://tech-knowledge-card`，由它执行 WebSearch 校验、内容构建、墨小颠配图生成（`image_gen` → `generated-images/`）。
2. **整理武侠风规范**：按下方「视觉风格详述」「风格专属内容要素」「组件清单」，把校验后的内容组织为四回目结构化文案 + 视觉/组件约束（供制卡使用，本技能不产出 HTML）。
3. **委托前端产品制卡**：发起 `@skill://顶尖的前端产品`，由其整理 PRD 并派发 `@skill://前端顶尖程序员` **从零实现**：
   - 移动端 HTML：`output/wuxia/{技术名词}/{技术名词}_{时间戳}_wuxia.html`，宽度固定 420px，CSS 内联自包含。
   - 网页版响应式 HTML：`output/wuxia/{技术名词}/{技术名词}_{时间戳}_wuxia_web.html`。
   - DOM 结构、CSS 类名、宣纸卷轴实现全部由前端程序员按本技能规范完成。
4. **截图推送**（可选）：委托 `@skill://wechat-publisher` 完成截图/切割/推送。
5. **Blog 同步**：委托 `@skill://tech-knowledge-card` / `@skill://blog-sync` 完成 Blog 静态站点同步。

```

```

## 视觉风格详述

- **基底**：宣纸卷轴质感（仿古纸底）
- **装饰**：木质卷轴顶/底杆（`class="scroll-rod"`）
- **配色**：朱砂红（`#c43b3b`）+ 墨色（`#2c2c2c`）+ 金色（`#d4b860`）
- **字体**：楷体/中宋字体
- **标签**：「武林秘籍」
- **章节**：用中文大写数字「壹·心法总纲」「贰·独门绝技」「叁·招式拆解」「肆·江湖险恶」四回目
- **署名**：朱砂红圆印章「{{AUTHOR_NAME}}印」（`class="chop-seal"`）

## 风格专属内容要素

### 章节命名

- 壹回 · 心法总纲 → 是什么？
- 贰回 · 独门绝技 → 为什么用？
- 叁回 · 招式拆解 → 怎么用？
- 肆回 · 江湖险恶 → 有什么问题？

### 幽默旁注

- 类型：「说书人插话」📜
- 位置：正文右侧/行间
- 语气：江湖说书人的俏皮包袱

### 运营策略适配（涨粉导向）

- **标题设计**：利用武侠"秘籍"感，制造好奇心和获得感
  - 推荐：好奇心（"秘籍"）、获得感（"绝技"）
  - 示例："Docker 武林秘籍：从入门到宗师"
- **封面设计**：在「武林秘籍」标签旁增加"独家""禁术"等标签
  - 示例："武林秘籍 · 禁术曝光 · Docker 心法"
- **开场白钩子**：使用模板 B（反直觉型），以"说书人"口吻呈现
  - 示例："说书人开讲 📜：你以为 Docker 是虚拟机？错了！这是容器界的九阳神功。"
- **金句提炼**：在章节标题或旁注中加粗可独立传播的金句
  - 示例："**容器化乃运维之道，隔离是术，镜像是器，网络是脉。**"
- **互动设计**：在「说书人插话」中埋彩蛋或求互动
  - 示例："各位侠士踩过哪些坑？留言区切磋 📜"
- **结尾行动召唤**：在朱砂红圆印章旁加关注引导
  - 示例："关注「{{AUTHOR_NAME}}」，下回讲 Kubernetes 内力心法"

### 特有组件

- 武功心法框（类比框，四角朱砂点缀）
- 独门绝技列表（⚔🛡⚡🔥 图标）
- 武林对决对比表（VS 双栏，江湖对决感）
- 招式拆解蓝图（武功招式风格架构图）
- 口诀代码块（深底金字，像秘籍口诀）
- 走火入魔警告（☠）+ 前辈指点（☯）
- 火焰分隔符（★ ☆ ★）

## 结构化内容产出（供前端程序员实现模板）

本技能输出的不是 HTML，而是下面各区块的**结构化内容**。前端产品据此整理 PRD，前端程序员据此实现 HTML 各区域：

1. **封面区**：「武林秘籍」标签 + 技术名词标题（朱砂高亮关键字）+ 剑客插图
2. **壹回 · 心法总纲**：一句话定义 + 生动类比（武功心法框）+ 修行者插图
3. **贰回 · 独门绝技**：3-5 个优势点（⚔🛡⚡🔥 标记）+ 武林对决 VS 表 + 轻功侠客插图
4. **叁回 · 招式拆解**：招式蓝图 + 最小可运行口诀（深底金字代码块）+ 命令招式速查
5. **肆回 · 江湖险恶**：2-4 个 ☠ 走火入魔警告 + ☯ 前辈指点忠告 + 持盾护卫插图
6. **说书人插话** 📜：幽默旁注分散在各回
7. **参考文献**：列出 2+ 个权威来源
8. **页末**：朱砂红圆印章「{{AUTHOR_NAME}}印」+ 抱拳告辞插图

## HTML 制卡委托（本技能不内置模板）

> **本技能不再直接生成 HTML，也不提供内置 `.html` / `.css` 模板文件**。HTML 制卡拆解为：内容方（本技能 + `@skill://tech-knowledge-card`）→ 产品方（`@skill://顶尖的前端产品`）→ 编码方（`@skill://前端顶尖程序员`）。模板的 DOM 结构、CSS 类名、宣纸卷轴实现全部由前端程序员按本技能「视觉风格详述」与下方组件清单从零完成。
