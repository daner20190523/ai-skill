# Xiaohongshu Reading Cards - 小红书读书卡片生成

## 核心理念

将读书笔记转化为小红书**多图轮播帖**——5到6张卡片，HTML一次生成后切图。每张卡文字不超过30字。

**涨粉的底层逻辑不是"好看"，是"有用+有观点+记得住是谁"**：

```
文章 -> 情绪核 + 利他价值 + 争议观点 -> 卡片内容分配(5-6张) -> 1个HTML -> Pillow切图 -> N张PNG
       |             |            |
       v             v            v
    Card 1-3      Card 5       Card 4(争议钩子)
    情绪共鸣      延伸书单      评论区战场
```

> **红线：存图高 != 涨粉多。** 小红书读书赛道点赞6万涨粉100的案例极多。涨粉需要三样东西同时到位：(1)卡片有辨识度（谁发的）(2)内容有信息增量（不止是读后感）(3)用户有回来的理由（系列追踪）。本技能强制产出这三个维度的内容。

- 用户提到"小红书""推文""卡片""首图""切图""涨粉""读书笔记"
- 用户指定一篇文章/一本书要做成小红书内容
- 用户说"帮我发小红书""做成小红书图文"

## 违规词红线（自动审 + 手动审）

生成内容后必须逐卡扫描以下词类，命中立即替换：

### 绝对禁止词

| 类别 | 禁止词示例 | 替代 |
|------|-----------|------|
| **极限词** | 最/第一/唯一/绝对/顶级/极/百分百/全网/彻底/完全 | 很/挺/比较/相对/蛮/不少 |
| **承诺词** | 保证/一定/肯定/必然/必定/绝对会 | 可能/也许/往往会/大概率 |
| **诱导词** | 加微信/私信我/关注我/扫码/进群 | 评论区聊聊/常来看看 |
| **比较拉踩** | 比XX好/碾压/秒杀/吊打 | 我更喜欢/对我更有用 |
| **虚假数据** | XX万人/全网爆火/疯了/炸了 | 不少人/挺多人/蛮受欢迎 |
| **医疗金融** | 治愈抑郁/赚/收益率/疗效 | 让人放松/情绪舒服 |

### 灰色地带词（慎用）

- "到哭""破防""逆天""杀疯了" -> 小红书可接受但影响调性，本技能**不用**
- "必读""必看""保姆级" -> 可用但控制频次，每篇<=1次
- emoji过度堆砌 -> 每张卡<=2个emoji

### 自动扫描

生成 content.md 后，对全文进行正则匹配，命中禁止词自动标记并给出替代建议。

## 四大涨粉引擎（优先级从高到低）

### 引擎 1：利他价值（权重最高）

小红书算法对"利他性内容"的赞藏加权比纯情绪笔记**高 200%**。每篇帖必须产出至少一项可被长期使用的价值：

| 利他形式 | 落点 | 示例 |
|---------|------|------|
| 延伸书单 | Card 5 | "读了《瓦尔登湖》，这3本你也会爱上" |
| 阅读方法论 | Card 5 或正文 | "我是这样啃完600页大部头的" |
| 分类推荐 | Card 5 | "如果你正在焦虑期，先读这本而不是那本" |
| 对比评测 | 正文 | "同是讲独处，《瓦尔登湖》和《孤独六讲》的区别" |

> **禁止产出纯读后感贴**。每篇必须包含至少1条利他信息——这是涨粉的第一性原理。

### 引擎 2：争议观点（驱动评论区）

评论区活跃 = 进入更大流量池。通用提问（"你喜欢哪本书？"）已失效。用**有态度的观点**替代：

```
❌ 通用："哪本书曾让你重新启动过？评论区聊聊"
✅ 争议："我一直觉得《瓦尔登湖》被误读最深——它不是教人隐居，是教人在漩涡里不丢自己。同意吗？"
✅ 选择："如果只能留一本哲学书，你留《瓦尔登湖》还是《存在与时间》？"
✅ 挑战："说实话，这本书第3-5章我看了3遍才懂。你们有读不下去的书吗？"
```

**争议钩子公式**：`我的鲜明观点 + 你同意/不同意？-> 二选一站队（降低评论门槛）`

### 引擎 3：人设锚点（存图->存人）

用户的关注动因是"这个人以后还能产出我喜欢的内容"，不是"这张图好看"。在卡片和正文中强制植入人设信号：

| 人设信号 | 落地方式 | 效果 |
|---------|---------|------|
| 阅读量标识 | 正文第一句 | "这是今年第47本，我终于敢聊这本书了" |
| 固定视觉符号 | 所有卡片左下角统一标识 | 用户滑到第二篇就能认出你 |
| 认知标签 | 简介/正文 | "专门拆哲学书的理科生""一年200本的非虚构沉迷者" |
| 系列归属 | 正文末尾 | "『改变我三观的24本书』第7期" |
| 署名 | Card N 右下角 | `-- {{AUTHOR_NAME}}` + 固定头像位 |

### 引擎 4：搜索截流（长尾流量）

小红书 **68% 流量来自搜索**。标题和正文必须嵌入用户会搜的关键词：

**标题关键词公式**：`[书名] + [人群/场景] + [结果/情绪] + [反常识角度]`

```
实例：
❌ "读完《原子习惯》的一些感悟"
✅ "拖延症晚期|这本《原子习惯》让我终于不用靠意志力早起"
```

**正文底部嵌入长尾词**（自然行文，不堆砌）：
```
"如果你也在找一本...的书" / "适合...的时候读" / "比...更...的一本"
```

## 卡片结构（5张标准版）

### Card 1 - 封面钩子卡

```
+--------------------+
|                    |
|    [意境底色]       |
|                    |
|   读 完《书名》     |
|   反常识一句话标题  |
|                    |
|   [装饰线]         |
|                    |
|   金句（竖排或大字号）|
|                    |
|   书名 · 作者       |
|   -- 智韵阁         |
|                    |
+--------------------+
```

**设计要求**：
- 文字不超过20字
- 标题采用 **"反常识标题公式"**：`读完《X》才知道，[一个打破默认认知的结论]`
- 深色/意境渐变背景（AI生成的bg_unified.png）
- 金句用衬线体大字，其余无衬线小字
- 大量留白，不堆元素
- 配色从书籍意境提取（瓦尔登湖->深蓝/墨绿/琥珀）
- **左下角嵌入固定人设标识**（小字 `-- {{AUTHOR_NAME}}`），每帖同一位置

### Card 2 - 痛点共鸣卡

```
+--------------------+
|                    |
|   你是不是也...？    |
|                    |
|   [极简3条共鸣点]   |
|                    |
|   说白了，XXXXXX    |
|                    |
|   [大面积留白]      |
|                    |
+--------------------+
```

**设计要求**：
- 文字不超过30字
- 浅色/暖色背景（纯色底色，从AI图色系提取）
- 痛点用列表式（左对齐），结尾一句扎心大白话居中
- emoji<=1个

### Card 3 - 金句翻译卡

```
+--------------------+
|                    |
|   [原文引用区块]    |
|   "......"         |
|                    |
|        ↓           |
|                    |
|   [大白话翻译]      |
|   说白了就是：XXXXXX |
|                    |
+--------------------+
```

**设计要求**：
- 文字不超过35字
- 上半部原文（衬线体/引号装饰）、下半部翻译（无衬线/加粗）
- 中间用一条细线或箭头分隔
- 纯色底色（中绿/中基调）+ 引用区深色微调
- **禁止出现"来自原文""原文第X行"等来源标注**——直接引用即可，不注出处

### Card 4 - 争议观点卡

```
+--------------------+
|                    |
|   这本书不是在讲XXX |
|                    |
|   它真正想说的是：   |
|                    |
|   [最终一句金句]    |
|                    |
|   ----------------- |
|                    |
|   [争议观点陈述]     |
|   同意吗？🤔        |
|                    |
|   -- 智韵阁         |
|                    |
+--------------------+
```

**设计要求**：
- 上部文字不超过25字，下部互动词不超过30字
- 开篇用反认知句式："不是在讲...真正想说的是..."
- **互动钩子必须是有态度的争议观点**，禁止通用提问
- 观点要具体到"这本书的某个解读角度"，格式：`我觉得[X]其实是[Y]，同意吗？`
- 底部署名 `-- {{AUTHOR_NAME}}`，建立人设锚点

### Card 5 - 利他延伸卡（强制）

```
+--------------------+
|                    |
|   如果喜欢这本      |
|                    |
|   这 3 本你也会爱上 |
|                    |
|   +--------------+ |
|   | 书1 · 1句话   | |
|   | 书2 · 1句话   | |
|   | 书3 · 1句话   | |
|   +--------------+ |
|                    |
|   "你读过哪本？🙋" |
|                    |
+--------------------+
```

**利他形式三选一**：

| 形式 | 卡片文案 | 适用场景 |
|------|---------|---------|
| **延伸书单** | "如果喜欢这本，这3本你也会爱上" | 通用 |
| **对比推荐** | "同是讲[主题]，这本和[某书]的区别" | 有明确竞品时 |
| **阅读方法** | "我是这样把这本书用起来的" | 方法论书籍 |

**设计要求**：
- 每本书 1 行：书名 + 一句差异化推荐理由（<=15字/本）
- 3 本书必须与主推书有**主题关联**（同题材/同作者/同情绪/同年代）
- 底部加低门槛互动："你读过哪本？""还有一本你推荐？"
- Cards 总数固定为 5 张，5 张是小红书轮播的黄金数量

> **可选 Card 6（极少用）**：仅当用户原文金句极多时，可把 Card 3 拆成两张（原文卡 + 翻译卡），总数 6 张封顶。

## HTML 美学规范

### 通用

| 规范项 | 要求 |
|--------|------|
| 卡片尺寸 | 1080x1440px（3:4） |
| 安全区 | 上下左右各留 80px 安全边距（文字不入安全区外） |
| 字体等级 | 大标题 56-72px / 正文 32-40px / 辅助 20-24px |
| 配色 | 每篇提取1个主色+1个强调色，不用超过3种颜色 |
| 行距 | 1.6-2.0（大字号用大行距） |
| 字重 | 关键句 700 / 正文 400-500 / 装饰 300 |
| 圆角 | 卡片底色无圆角（切图后无圆角需求） |
| 不要 | 不要阴影、不要边框、不要渐变文字、不要动画 |

### 排版原则

1. **少即是多**：每张卡最多3个信息层级
2. **不对称留白**：文字偏左或偏上，另一侧大留白
3. **字号对比**：最大字和最小字比例 >= 3:1
4. **垂直呼吸**：段落间留白 >= 字号的1.5倍
5. **不要居中堆砌**：避免所有文字都居中——会显廉价

### 配色方案（7套预设）

| 方案名 | 主色 | 强调色 | 适合书类 |
|--------|------|--------|---------|
| 瓦尔登湖 | `#1a2a3a` 深蓝黑 | `#c49a6c` 琥珀暖 | 哲学/自然/沉思 |
| 浮生六记 | `#f5f0e8` 暖米白 | `#8b4513` 赭石棕 | 散文/生活/中式 |
| 厌世者 | `#2d2d2d` 暗灰 | `#e8e8e8` 冷白 | 存在主义/批判 |
| 红楼梦 | `#8b1a1a` 朱砂红 | `#d4a574` 香槟金 | 古典文学 |
| 极简 | `#ffffff` 纯白 | `#1a1a1a` 纯黑 | 方法论/工具书 |
| 星辰 | `#0a1628` 深空蓝 | `#f0c040` 星光黄 | 科幻/宇宙/宏大叙事 |

## HTML 多卡片生成规范

### 硬性规则

#### A. 忠于原文——禁止杜撰

**所有卡片文字必须来自用户原文**。禁止自己编造"金句""翻译"或"一句扎心"。

```
用户原文 -> 提取关键句 -> 分配卡片位置 -> 排版
```

❌ 坏："你以为的孤独其实是自由"（杜撰）-> ✅ 好："原来人还可以这样生活"（原文第5行）

**文字来源优先级**：
1. 直接引用原文句子（最优先）
2. 原文句子的适度截断（保留语感）
3. 同一段落观点的极简浓缩（仍须还原原文说法）

#### B. 统一色调——首图AI + 其余纯色底色

**只有 Card 1 使用 AI 生成的意境背景图**（`bg_unified.png`），作为封面钩子的视觉锚点。

**Cards 2-5 使用从 AI 图提取的纯色底色**，从同一色系搭配出 4 张纯色/渐变卡片。禁止Cards 2-5也贴背景图。

```html
<!-- Card 1: AI 背景图 + 遮罩 -->
<div class="card" id="card-1" style="background-image: url('bg_unified.png')">
  <div class="overlay"></div>...
</div>

<!-- Cards 2-5: 纯色底色，从 AI 图色系提取 -->
<div class="card" id="card-2" style="background: linear-gradient(180deg, ...)">...</div>
<div class="card" id="card-3" style="background: #1a2822">...</div>
<div class="card" id="card-4" style="background: linear-gradient(180deg, ...)">...</div>
<div class="card" id="card-5" style="background: linear-gradient(180deg, ...)">...</div>
```

**CSS 全局颜色变量（从 AI 背景图取色，全篇统一）**：
```css
:root {
  --cream: #f5efe4;   /* 文字色 */
  --amber: #b8845c;   /* 强调色 */
  --dark:  #1a2a24;   /* 深基底色 */
  --mid:   #1f3028;   /* 中绿底色 */
  --warm:  #2a2218;   /* 暖棕底色 */
}
```

**卡片底色策略**：

| 卡片 | 底色来源 | 样式 |
|------|---------|------|
| Card 1 封面 | AI 生成图片 `bg_unified.png` | background-image + 半透明渐变遮罩 |
| Card 2 共鸣 | 色系提取暖棕 `--warm` | `background: linear-gradient(...)` 柔和渐变 |
| Card 3 金句 | 色系提取中绿 `--mid` | `background: #1f3028` 纯色 |
| Card 4 争议 | 色系提取深基 `--dark` | `background: linear-gradient(...)` 深渐变 |
| Card 5 利他 | 色系提取暖-中渐变 | `background: linear-gradient(...)` 暖底提亮 |

### HTML 结构

```html
<div class="card-stack">
  <div class="card" id="card-1"><div class="overlay"></div>...</div>
  <div class="card" id="card-2"><div class="overlay"></div>...</div>
  <div class="card" id="card-3"><div class="overlay"></div>...</div>
  <div class="card" id="card-4"><div class="overlay"></div>...</div>
  <div class="card" id="card-5"><div class="overlay"></div>...</div>
</div>
```

**CSS 关键约束**：
- 每张 `.card` 宽高固定 `width:1080px; height:1440px`
- 卡片间**无间距**（margin:0），紧贴堆叠
- `.card-stack` 无 padding、无 gap
- body margin:0，完全无白边
- `box-sizing: border-box` 全局
- 所有文字使用 `px` 单位，不随视口缩放
- **Card 1 使用 background-image，Cards 2-5 使用纯色/渐变 background**

### 设计检查清单

生成 HTML 前检查：
- [ ] 每张卡文字来自**用户原文**，无杜撰
- [ ] **Card 1 用 AI 背景图**，Cards 2-5 用同色系纯色底色
- [ ] 配色全篇统一（:root 变量从 AI 图取色）
- [ ] **Card 1 左下角有人设标识**（`-- {{AUTHOR_NAME}}`）
- [ ] **Card 4 争议观点**不是通用提问，有明确立场
- [ ] **Card 5 利他价值**存在且不少于3本书/3条信息
- [ ] 无极限词、无承诺词、无诱导词
- [ ] emoji每卡<=1个
- [ ] 卡片间无间距
- [ ] 1080x1440 精确尺寸

## 切图流程

使用脚本 `scripts/html_slice.py`：

```bash
python scripts/html_slice.py output/{书名}_{时间戳}/cards.html --card-height 1440
```

**内部逻辑**：
1. Playwright 渲染完整 HTML -> 全页截图（1080 x 1440xN）
2. Pillow 读取截图，按 1440px 步长竖向裁切
3. 输出 `card_01.png` ~ `card_0N.png` 到同目录
4. **同时保留完整长图 `full_page.png`** 作为预览/归档

切完后验证每张图尺寸均为 1080x1440。

## 正文公式（content.md）—— 150-300字，禁止60字

小红书算法考核**深读时长**。60字2秒读完被判"快餐内容"，推入低层级流量池。正文需要写得让人停留至少 15 秒。

### 正文结构

```markdown
👋 [人设开场，1句话]：我是XXX，今年第N本/专门读XX类型/最近在补XX系列

📖 [书名]——[反常识标签，嵌入搜索关键词]

[空行]

[第一段 30-50字] 这本书最戳我的一个点（不剧透核心，只讲情绪钩子）
```

## 语气基调：暖中带刺

小红书核心读书用户是 **18-35 岁女性**。她们不是来找冷嘲热讽的——她们需要"被理解、被戳中、被启发"。语气底盘必须是**温暖共情的**，锋芒只是用来制造传播的。

### 四层语气规则

| 层级 | 语气 | 落地位置 | 写法 |
|------|------|---------|------|
| **底盘：共情** | "我懂你"的温暖感 | Card 2 / 正文第一段 | "你是不是也...""我当初翻开是因为..." |
| **中层：真诚** | 不自嗨、不止于漂亮话 | Card 3 / 正文第二段 | 引用原文后加"这段话我看了三遍才有点明白" |
| **锋芒：观点** | 有立场的判断（不是嘲讽） | Card 4 争议钩子 | "这本书被严重高估的其实是第X章——它真正厉害的是..." |
| **收尾：邀请** | 把评判权还给读者 | Card 4/5 底部 | "你同意吗？""你觉得呢？"（真问，不是套路） |

### 禁用语气

- ❌ 说教/居高临下："你应该读这本书""这本书会改变你"
- ❌ 高冷/刻薄："看不懂是你傻""大多数人根本不懂"
- ❌ 虚假激动："绝了！""谁懂啊！！""天哪！"
- ❌ 空洞鸡汤："生活不止眼前的苟且""做自己就好"

### 替换对照

| 旧（冷锋） | 新（暖刺） |
|-----------|-----------|
| "一刀切进肉里" | "一句话把我钉在原地" |
| "可惜大多数人没看懂" | "我也是读第二遍才注意到这个细节" |
| "说白了就是" | "翻译一下就是——"（破折号比"说白了"更柔和） |
| "不是解决了，是发现大家都一样" | "原来不只我一个人这样想" |

## 系列化内容策略

单篇爆款不会涨粉。**系列是涨粉的唯一加速器**。每篇帖子必须归属于一个明确的内容系列。

### 系列命名公式

```
[数量] + [角度/态度] + [品类]
```

示例：
- `『改变我三观的24本书』`（数量+结果+品类）
- `『适合焦虑期反复读的10本』`（场景+数量+品类）
- `『被书名耽误的好书』`（态度+品类）

### 正文尾部模板

```markdown
📌 同系列已拆：「书名1」「书名2」「书名3」-> @主页查看『{{SERIES_NAME}}』全集
```

### 系列落地的 3 个条件

1. **有总数**：24本/10本/5本——让用户知道要追多久
2. **有规律**：每周一篇/每两天一篇——让用户知道什么时候回来
3. **有进度**：正文标注"第X期"——让用户感知连续感

## 十一不原则（常见错误避免）

1. **不要杜撰文字**：卡片所有文字必须来自用户原文，禁止自己编金句
2. **不要 4 卡同一底图**：Card 1 用 AI 图，Cards 2-5 用同色系纯色底色
3. **不要缺利他价值**：Card 5 强制产出延伸书单/对比/方法，禁止纯读后感
4. **不要通用互动**：Card 4 必须是有观点的争议钩子，禁用"你喜欢哪本书？"
5. **不要缺人设锚点**：Card 1 左下角和 Card 4/5 底部必须有 `-- {{AUTHOR_NAME}}`
6. **不要文字塞满**：卡片竞争力是"高级感排版"，不是信息密度
7. **不要正文太短**：最低 120 字，150-300 字是黄金区间
8. **不要违规词**：宁可用"挺""比较"也不用"最""第一"
9. **不要居中堆砌**：左对齐+不对称留白 = 杂志感，全居中 = 廉价感
10. **不要只发单篇**：正文末尾标注系列名称，给用户"还有下一期"的期待
11. **不要标注来源**：所有卡片文字**禁止**出现"来自原文""原文第X行""本文引用自"等来源标注，直接呈现引文即可
